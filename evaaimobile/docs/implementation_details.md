# Реализация приложения EVA AI (Mobile)

Данный документ описывает архитектуру и ключевые технические решения, реализованные в мобильном приложении на React Native.

## 📁 Структура проекта (src)

*   **`screens/`** — Экраны приложения.
    *   `ChatScreen.js`: Основной компонент чата. Реализует сложную логику взаимодействия с ИИ.
    *   `MainScreen.js`: Главный экран со списком персонажей.
    *   `CharacterScreen.js`: Профиль персонажа.
    *   `FavoritesScreen.js`: Экран со списком избранных персонажей.
*   **`services/`** — Слой взаимодействия с API.
    *   `chatService.js`: Отправка сообщений, получение истории, отправка подарков и проверка баланса.
    *   `apiClient.js`: Настроенный экземпляр Axios с базовым URL и интерцепторами.
    *   `pushNotificationService.js`: Логика работы с Firebase Cloud Messaging (FCM).
*   **`store/`** — Управление состоянием (Zustand).
    *   `authStore.ts`: Хранение данных пользователя, токена и текущего баланса.
    *   `favoritesStore.ts`: Управление списком избранных персонажей.
*   **`components/`** — Общие UI-компоненты (модальные окна, кнопки).

## 💬 Реализация чата (ChatScreen.js)

Чат построен на базе библиотеки `react-native-gifted-chat` (v2.4.0) с глубокой кастомизацией.

### Ключевые фичи:

1.  **Имитация «человеческой» печати**:
    *   При получении ответа от сервера, новые сообщения от ИИ добавляются в чат не мгновенно, а по очереди.
    *   Используется расчет задержки (typing speed) на основе длины текста.
    *   Отображается индикатор `isTyping` во время пауз.

2.  **Очистка данных OpenRouter**:
    *   Реализован парсер в `formatHistory`, который отсекает технические логи OpenRouter (JSON-отчеты о токенах и стоимости) и выводит пользователю только чистый текст сообщения.

3.  **Стабильность ID сообщений**:
    *   Используются стабильные ID, привязанные к серверным данным, что предотвращает дублирование сообщений при обновлении истории.
    *   Использование `useRef` для `messages` гарантирует актуальность данных при асинхронных операциях.

4.  **Исправление React 19 Warning**:
    *   Кастомный `renderMessage` явно передает `key` до деструктуризации пропсов, что устраняет предупреждение о "key prop spread".

## ⌨️ Работа с клавиатурой (Android)

Для стабильной работы поля ввода на Android реализованы следующие настройки:

*   **AndroidManifest.xml**: Установлен параметр `android:windowSoftInputMode="adjustResize"`.
*   **KeyboardAvoidingView**: Используется `behavior="height"` с `keyboardVerticalOffset={100}`, что гарантирует видимость поля ввода над клавиатурой на большинстве устройств, включая планшеты.
*   **SafeArea**: Управление отступами происходит через `useSafeAreaInsets`, что исключает конфликты с системными панелями на устройствах с "челками".

## 🌐 Сетевое взаимодействие

*   **Endpoint Slashes**: В `characterService.js` используется обязательный завершающий слэш для `/characters/`, чтобы избежать лишних 307 редиректов.
*   **Обновление баланса**: Баланс синхронизируется после каждого отправленного сообщения или подарка через `updateBalance` в глобальном сторе.

## ❤️ Избранное (Favorites)

Реализована система закладок для персонажей:
*   **Store**: Используется `favoritesStore.ts` для персистентного хранения списка ID избранных персонажей.
*   **UI**: Иконка сердца на карточках в `MainScreen` и в профиле `CharacterScreen`.
*   **Экран**: Отдельный `FavoritesScreen` для быстрого доступа к любимым героям.

## 🔔 Пуш-уведомления (Firebase FCM)

Система уведомлений реализована на базе Firebase Cloud Messaging:
1.  **Регистрация**: При входе в аккаунт приложение запрашивает разрешение и получает FCM-токен.
2.  **Синхронизация**: Токен отправляется на сервер через `POST /api/user/push-token` для привязки к пользователю.
3.  **Обработка**: Сервис `pushNotificationService.js` обрабатывает сообщения как в активном режиме (Alert), так и при клике по уведомлению из системной шторки.
