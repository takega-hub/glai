# Инструкция по работе с миграциями базы данных

Этот документ описывает процесс создания и применения миграций для базы данных проекта EVA AI.

## Общий принцип

Мы используем "чистые" SQL-миграции. Каждый раз, когда требуется изменить схему базы данных (добавить таблицу, колонку, индекс и т.д.), создается новый `.sql` файл в директории `api/migrations`.

Основная схема базы данных определена в файле `api/schema.sql`.

---

## Способ 1: Полное пересоздание базы данных (ОПАСНО: все данные будут удалены!)

Этот способ используется только в крайних случаях, когда база данных находится в несогласованном состоянии или требуются глобальные изменения схемы.

1.  **Выполнение скрипта:**
    Выполните следующую команду в терминале. Она удалит все таблицы и воссоздаст их заново из `api/schema.sql`, а затем применит все существующие `.sql` миграции.

    ```bash
    # Копируем основной файл схемы в контейнер
    docker cp api/schema.sql eva_ai_db:/tmp/schema.sql

    # Копируем папку с миграциями
    docker cp api/migrations eva_ai_db:/tmp/migrations

    # Выполняем полное пересоздание
    docker-compose exec -T db bash -c "\
        psql -U glai -d eva_ai -c \"DO \\\$\$ DECLARE r RECORD; BEGIN FOR r IN (SELECT tablename FROM pg_tables WHERE schemaname = 'public') LOOP EXECUTE 'DROP TABLE IF EXISTS ' || quote_ident(r.tablename) || ' CASCADE'; END LOOP; END \\\$\$;\" && \
        psql -U glai -d eva_ai -f /tmp/schema.sql && \
        for f in /tmp/migrations/*.sql; do psql -U glai -d eva_ai -f \"\$f\"; done"
    ```

---

## Способ 2: Создание и применение новой миграции (безопасный способ)

Это стандартный способ внесения изменений в базу данных.

1.  **Создайте новый SQL-файл:**
    В директории `api/migrations` создайте новый файл с осмысленным названием, например: `20250403_add_user_bio.sql`.

2.  **Напишите SQL-код:**
    Внутри файла напишите SQL-команду, которую вы хотите выполнить. Например:

    ```sql
    -- Добавляем колонку "биография" в профили пользователей
    ALTER TABLE user_profiles ADD COLUMN bio TEXT;
    ```

3.  **Примените миграцию:**
    Скопируйте ваш новый файл в контейнер с базой данных и выполните его с помощью `psql`.

    ```bash
    # Замените <имя_файла> на имя вашего файла миграции
    MIGRATION_FILE=20250403_add_user_bio.sql

    # Копируем файл
    docker cp api/migrations/$MIGRATION_FILE eva_ai_db:/tmp/$MIGRATION_FILE

    # Выполняем миграцию
    docker-compose exec -T db psql -U glai -d eva_ai -f /tmp/$MIGRATION_FILE
    ```

Этот подход позволяет вносить изменения постепенно и безопасно, не затрагивая существующие данные.
