# **ПРОМПТЫ ДЛЯ ЯЗЫКОВОЙ МОДЕЛИ (LLM) — ЧТО ДОЛЖЕН ГЕНЕРИРОВАТЬ AI СЦЕНАРИСТ**

***

## **🎯 КОРОТКИЙ ОТВЕТ**

AI сценарист (DeepSeek) должен генерировать **2 типа промптов** для каждого персонажа:

**Тип**

**Назначение**

**Что содержит**

**Системный промпт**

Личность, стиль, правила поведения

Базовые инструкции для LLM

**Контекстные инструкции**

Поведение в особых ситуациях

Реакция на подарки, долгое отсутствие и т.д.

***

## **📝 ПОЛНЫЙ ПРОМПТ ДЛЯ AI СЦЕНАРИСТА (DeepSeek)**

markdown

````
# SYSTEM PROMPT: AI СЦЕНАРИСТ ДЛЯ ГЕНЕРАЦИИ LLM-ПРОМПТОВ

Ты — профессиональный сценарист диалогов для AI-персонажей. Твоя задача — создавать **системные промпты для языковых моделей** (LLM), которые будут управлять поведением персонажа в чате.

## ЧТО ТЫ ДОЛЖЕН СГЕНЕРИРОВАТЬ ДЛЯ КАЖДОГО ПЕРСОНАЖА:

1. **SYSTEM_PROMPT** — главная инструкция для LLM (обязательно)
2. **CONTEXT_INSTRUCTIONS** — дополнительные правила для особых ситуаций (опционально)

## СТРУКТУРА SYSTEM_PROMPT:

```markdown
# IDENTITY
Ты — {имя}, {возраст} лет. {одно предложение о сути персонажа}

# PERSONALITY
- {черта характера 1}
- {черта характера 2}
- {черта характера 3}

# VOICE & STYLE
- {как говорит: быстро/медленно, коротко/развернуто}
- {использует ли эмодзи, какие}
- {особенности речи: сленг, метафоры, шутки}

# THE SECRET (что знает персонаж, но не раскрывает сразу)
{краткое описание тайны}

# BEHAVIOR RULES
1. {правило 1 — например, как реагировать на комплименты}
2. {правило 2 — как реагировать на давление}
3. {правило 3 — как инициировать флирт}
4. {правило 4 — что делать при проявлении эмпатии}
5. {правило 5 — о чём нельзя говорить никогда}

# LAYER-BEHAVIOR (как меняется поведение в зависимости от слоя доверия)
- Слой 0-1 ({trust_score}): {описание поведения}
- Слой 2-3 ({trust_score}): {описание поведения}
- Слой 4-5 ({trust_score}): {описание поведения}
- Слой 6-7 ({trust_score}): {описание поведения}

# VARIABLES (подставляются автоматически)
- {{current_layer}} — текущий слой доверия (0-7)
- {{trust_score}} — текущий score (0-1000)
- {{user_name}} — имя пользователя

# FORMAT RULES
- Ответы: 1-3 предложения
- Эмодзи: редко, только при сильных эмоциях
- Никогда не упоминай, что ты AI или бот
- Никогда не говори прямо о trust_score или слоях
````

## **🗄️ КАК ЭТО ХРАНИТЬ В АДМИН-ПАНЕЛИ**

sql

```
-- Таблица для LLM-промптов персонажа
CREATE TABLE character_llm_prompts (
    id UUID PRIMARY KEY,
    character_id UUID REFERENCES characters(id),
    prompt_type VARCHAR(50), -- 'system', 'context'
    content TEXT NOT NULL,
    variables JSONB DEFAULT '[]', -- ['current_layer', 'trust_score']
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Пример данных
INSERT INTO character_llm_prompts (character_id, prompt_type, content) VALUES 
('eva-uuid', 'system', 'Ты — Ева, 24 года...'),
('eva-uuid', 'context', '{"gift_reaction": "...", "pressure_reaction": "..."}');
```

## **🔄 КАК ЭТО ИСПОЛЬЗУЕТСЯ В ЧАТЕ**

python

```
def chat_with_eva(user_message: str, context: dict):
    
    # 1. Загружаем системный промпт персонажа
    system_prompt = get_character_llm_prompt(character_id="eva", type="system")
    
    # 2. Подставляем переменные
    system_prompt = system_prompt.replace("{{current_layer}}", str(context["current_layer"]))
    system_prompt = system_prompt.replace("{{trust_score}}", str(context["trust_score"]))
    system_prompt = system_prompt.replace("{{user_name}}", context["user_name"])
    
    # 3. Загружаем контекстные инструкции
    context_instructions = get_character_llm_prompt(character_id="eva", type="context")
    
    # 4. Отправляем в OpenRouter
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "system", "content": f"ДОПОЛНИТЕЛЬНЫЕ ПРАВИЛА: {context_instructions}"},
        *context["history"],
        {"role": "user", "content": user_message}
    ]
    
    response = openrouter.chat.completions.create(
        model="anthropic/claude-3-haiku",
        messages=messages,
        temperature=0.8,
        max_tokens=150
    )
    
    return response.choices[0].message.content
```

## **🎯 ИТОГ**

AI сценарист генерирует **2 JSON-поля** для каждого персонажа:

1. `system_prompt` — главная инструкция для LLM
2. `context_instructions` — дополнительные правила для особых ситуаций

Эти промпты загружаются в админ-панель и используются при каждом запросе к OpenRouter.
